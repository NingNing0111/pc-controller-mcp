# AI 可操作截图网格系统设计方案（Rust）

## 一、目标

构建一套用于 AI 操作电脑的截图网格系统，实现：

- 精确定位屏幕位置
- 降低坐标误差
- 提供结构化输入给 AI
- 支持递归细化（类似人类视觉聚焦）
- 与鼠标控制系统无缝集成

---

## 二、整体架构

```

截图 → 网格生成 → 图像标注 → 结构化数据 → AI决策 → 鼠标执行

```

---

## 三、坐标系统设计

### 1. 基础坐标系

- 原点：左上角 (0, 0)
- X轴：向右
- Y轴：向下

### 2. 坐标类型

| 类型       | 用途     |
| ---------- | -------- |
| 像素坐标   | 实际点击 |
| 归一化坐标 | AI输入   |

```

normalized_x = x / width
normalized_y = y / height

```

---

## 四、网格系统设计（核心）

## 4.1 分层网格结构（Hierarchical Grid）

### Level 1（粗网格）

- 推荐：6 × 4
- 标识方式：A1 ~ D6

```

A1 A2 A3 A4 A5 A6
B1 B2 B3 B4 B5 B6
C1 C2 C3 C4 C5 C6
D1 D2 D3 D4 D5 D6

```

---

### Level 2（细网格）

- 对任意格子进行 3 × 3 划分
- 标识方式：B3-a1 ~ B3-c3

```

a1 a2 a3
b1 b2 b3
c1 c2 c3

```

---

### 最终定位格式

```

<Level1>-<Level2>

示例：
B3-b2
A1-c3

```

---

## 五、数据结构设计

### GridCell

```rust
struct GridCell {
    id: String,
    x1: u32,
    y1: u32,
    x2: u32,
    y2: u32,
}
```

---

### ScreenData

```rust
struct ScreenData {
    width: u32,
    height: u32,
    grid_cells: Vec<GridCell>,
}
```

---

### ClickAction

```rust
struct ClickAction {
    action: String,
    target: String,
}
```

---

## 六、网格生成算法

### 6.1 一级网格生成

```rust
fn generate_grid(width: u32, height: u32, rows: u32, cols: u32) -> Vec<GridCell>
```

计算方式：

```
cell_width = width / cols
cell_height = height / rows
```

---

### 6.2 二级网格生成

```rust
fn subdivide(cell: &GridCell) -> Vec<GridCell>
```

规则：

- 每个 cell 划分为 3×3
- 子ID：<parent>-a1 ~ c3

---

### 6.3 坐标解析

```rust
fn get_center(cell: &GridCell) -> (u32, u32)
```

```
center_x = (x1 + x2) / 2
center_y = (y1 + y2) / 2
```

## 八、AI交互协议

### 输入（Prompt）

```
屏幕被划分为网格：

一级网格：A1 ~ D6
每个格子可以细分为：
a1 ~ c3

请返回你要点击的位置，例如：
B3-b2

只返回ID，不要解释
```

---

### 输出（AI）

```json
{
  "action": "click",
  "target": "B3-b2"
}
```

---

### 执行流程

1. 解析 target
2. 定位 GridCell
3. 计算 center
4. 执行鼠标点击

---

## 九、递归放大机制（关键）

### 流程

```
1. 全屏截图
2. AI选择 Level1（如 B3）
3. 截取 B3 区域
4. 对该区域重新划网格
5. AI选择 Level2（如 b2）
6. 执行点击
```

---

### 优势

- 提高精度
- 降低视觉复杂度
- 模拟人类操作方式

## 十二、注意事项

### DPI缩放

- 截图坐标 ≠ 鼠标坐标
- 必须统一坐标空间

---

### 多显示器

- 需要统一虚拟坐标系
- 或限制单屏操作

---

### 性能优化

- 避免全屏频繁截图
- 使用局部截图
